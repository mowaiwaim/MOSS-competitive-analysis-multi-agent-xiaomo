# 第五版报告形态目标规格

参考标杆：`ChatGPT_vs_DeepSeek_competitive_analysis_2026-05-27.pdf`。

本规格用于指导后续报告生成、前端展示和 PDF 导出升级。目标不是复制固定模板，而是让系统稳定产出“分析全面、证据充分、图表清晰、可追溯”的竞品分析报告。

## 标杆报告结构

1. 封面与核心结论
   - 标题、生成日期、资料核验日期、资料来源范围。
   - 4 个核心结论卡：价格结论、产品结论、评价结论、风险结论。
   - 一段明确的综合判断，直接回答“谁更适合什么场景”。
   - 报告范围表，说明本次覆盖的功能、价格、评价、SWOT 和风险边界。

2. 执行摘要
   - 用三列表格比较竞品：竞品 A、竞品 B、分析判断。
   - 覆盖产品定位、功能完整度、价格、用户评价、战略风险。
   - 页面内附功能评分热力条，不只输出文字。

3. 方法与来源可靠性
   - 按信息类型列出主要来源、可靠性和使用方式。
   - 区分官方事实、第三方评价、新闻风险和模型/参数声明。
   - 明确边界条件：价格、地区、评价样本、性能测试和合规审查。

4. 产品与功能对比
   - 功能域表：日常问答、文件/数据分析、多模态、Agent、企业治理、开放/自部署等。
   - 每行必须有“竞品含义”，避免只罗列功能。
   - 可视化包括雷达图和竞争定位图。

5. 定价与成本对比
   - 订阅/团队价格表。
   - API 标价表。
   - 成本指数柱状图和对数尺度价格图。
   - 必须展示计算口径，例如 “DeepSeek V4-Pro 输出价约为 GPT-5.5 的 2.9%”。

6. 用户评价与口碑
   - 区分 G2、App Store、Trustpilot 等不同平台样本。
   - 区分功能满意度、售后投诉、企业采购风险。
   - 评价不能只看星级，必须解释样本偏差和平台差异。

7. SWOT
   - 每个竞品独立 SWOT 卡片，不共用模板句。
   - 每个象限至少 3 条要点。
   - 优势、弱点、机会、威胁必须来自已采集证据或明确标为推断。

8. 场景化采购/使用建议
   - 后续恢复建议模块时，不做泛化“建议与结论”。
   - 应改为“场景化决策矩阵”：场景、推荐优先级、理由、落地动作。
   - 建议必须绑定来源、评分口径和不确定性。

9. 关键洞察与风险控制
   - 3 到 5 条关键洞察，每条解释业务含义。
   - 风险控制表：风险、竞品 A 控制建议、竞品 B 控制建议。

10. 附录与来源列表
    - 价格与事实备注。
    - 来源编号、来源名称、用途、URL。
    - 报告正文引用 `[S1] [S2]`，可回溯到来源表。

## 我们当前系统的差距

- 当前报告是网页正文型，缺少封面式结论卡、方法页、来源可靠性页和附录。
- 当前 `claims` 可以溯源，但还没有稳定生成 `source_catalog`、`methodology`、`decision_matrix`、`risk_controls` 等报告级结构。
- 当前可视化较弱，缺少雷达图、定位图、成本指数柱状图、热力条和 SWOT 彩色卡片。
- 当前定价和评分没有统一计算口径，无法稳定生成“相差多少倍”“成本指数”等量化结论。
- 当前建议模块仍暂时移除；未来只能以“场景化决策矩阵”形式恢复，不能再输出泛化建议。

## 后续实现路线

### P0：数据与结构先补齐

- 在报告内容 JSON 增加：
  - `executive_cards`
  - `methodology`
  - `source_reliability`
  - `feature_scores`
  - `pricing_comparison`
  - `review_summary`
  - `decision_matrix`
  - `risk_controls`
  - `source_catalog`
- 采集 Agent 的搜索计划要固定覆盖：
  - 官方价格页
  - 官方产品/功能页
  - 官方 API/开发者文档
  - 官方隐私/企业/安全页面
  - G2/App Store/Trustpilot 等评价
  - 新闻/监管/风险来源
- 分析 Agent 必须先写结构化字段，再派生图表数据和报告结论。

### P1：网页报告视觉升级

- 报告页改为类 PDF 的长报告布局：
  - 封面结论卡
  - 摘要矩阵
  - 功能热力条
  - 雷达图
  - 竞争定位图
  - 价格柱状图
  - SWOT 彩色卡片
  - 来源附录
- 图表可先用前端 SVG/Canvas 实现，数据来自报告 JSON，不在前端编造。

### P2：PDF 导出

- 使用 ReportLab 生成 PDF，因为当前环境已经具备 `reportlab`。
- 新增接口：
  - `GET /api/tasks/<task_id>/report/pdf`
- PDF 必须复用同一份报告 JSON，避免网页和 PDF 两套逻辑不一致。

### P3：质量门槛

- 没有来源的关键结论不得进入核心结论卡。
- 价格、API、政策、评分必须展示核验日期。
- 图表必须展示口径说明。
- 第三方评价必须标注样本偏差。
- 场景化建议必须绑定证据和不确定性。
